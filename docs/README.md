# docs/ — 투 트랙 기록

이 폴더는 두 가지를 담는다. **섞지 않는다.**

| 폴더 | 뭐가 들어가나 | 누가 쓰나 | 언제 |
|---|---|---|---|
| `decisions/` | **왜 이렇게 짰나.** 이 프로젝트 한정. 날짜 있음 | 개발 세션이 자동으로 | 개발 세션 끝날 때 |
| `concepts/` | **여기 쓰인 개발 지식.** 다음 프로젝트에도 씀. 날짜 없음 | 학습 세션이 질문 받고 | 궁금할 때 |

`program-ops.md`는 둘 다 아니다. 진행 담당자용 운영 문서다.

## 왜 나눴나

**결정**은 프로젝트에 묶여 있고 유통기한이 있다. "1인 개발이라 게임을 4개로 줄였다"는
이 MT가 끝나면 죽는 정보다.

**개념**은 어디서든 산다. "판별 유니온이 뭐고 언제 쓰나"는 다음 프로젝트에도, 면접에서도,
5년 뒤에도 쓴다. 이게 개인 목표에 해당하는 자산이다.

섞으면 둘 다 망한다. 결정 기록에 개념 설명이 끼면 길어져서 아무도 안 읽고,
개념 노트에 이 프로젝트 사정이 끼면 다음 프로젝트에서 못 쓴다.

## 트랙 돌리는 법

**터미널 두 개. 같은 폴더. `claude` 두 번.** 세션이 나뉘면 컨텍스트가 나뉜다.

### 개발 세션
**평소대로 쓴다. 아무것도 안 해도 된다.**

의미 있는 판단을 내렸으면 Claude가 시키지 않아도 `decisions/`에 남기고, 새로 등장한 개념은
`concepts/`에 **스텁**으로 꽂는다. CLAUDE.md "세션 기록 — 시키지 않아도 한다"에 박혀 있다.

**왜 자동이냐:** 기억해서 명령을 쳐야 하면 그 트랙은 죽는다. 세션은 깔끔하게 안 끝난다 —
피곤해서 노트북을 덮는다. 그때 재료가 이미 남아 있어야 한다.

`/logbook`은 **수동 강제용**으로 남아 있다. 자동으로 안 남긴 것 같을 때, 또는
"지금 이거 기록해"라고 콕 집을 때.

### 학습 세션
`/learn <질문>`.
→ 코드를 **절대 수정하지 않는다.** 답한 내용은 `concepts/`에 남는다. 대화로 끝내지 않는다.

## 스텁이 학습 큐다

개발 세션은 개념 노트를 **채우지 않는다.** 이름과 한 줄만 꽂는다.
설명을 미리 다 쓰면 안 읽을 문서를 쓰는 데 개발 시간을 태우는 것이다.

그래서 `concepts/`에는 항상 안 채워진 스텁이 남아 있다. **그게 정상이고, 그게 목록이다.**
하나 골라서 `/learn`으로 물어보면 채워진다. 밀려 있어도 개발은 안 막힌다.

**스텁은 `#스텁` 태그로 찾는다.**
- 옵시디언: 오른쪽 사이드바 태그 창에 트리로 뜬다. 클릭하면 목록
- 태그가 안 잡히면: `Ctrl+Shift+F` → `스텁` 검색. 같은 결과가 나온다
- `/learn`이 노트를 채우면 태그를 뗀다. **그 숫자가 줄어드는 게 진행률이다**

> 태그를 쓰는 이유: 스텁도 실제 파일이라 위키링크가 정상 해결된다. 그래서 그래프에선
> 채워진 노트와 구분이 안 된다. 발견은 태그가 맡는다.

## 티어 — 학습이 개발을 막지 않게 하는 장치

```
스텁
  ├ 당일   ← MT 당일 8시 35분에 이게 없으면 못 고친다. 하드 데드라인
  └ 성장   ← 마감 없음. 개인 목표
```

**게이트는 없다. 영원히.** "이거 배우고 넘어가세요"를 만들면 학습이 숙제가 되고,
숙제가 되면 죽는다. 티어는 막는 게 아니라 표시만 한다.

### 왜 대부분이 `성장`인가

**코드는 Claude가 짠다.** 사용자가 브랜드 타입을 몰라서 프로젝트가 망하는 경로는 없다.
지금까지 이 프로젝트를 살린 판단들(게임 자르기, 예비 정원, 리허설 우선순위)은 전부
**제품 판단**이었고 CS 지식이 0만큼 들어갔다.

지식이 하중을 받는 지점은 딱 하나다 — **MT 당일 밤, 서버가 죽었고, Claude는 거기 없다.**

### `당일` 배정 기준

Claude의 취향이 아니라 CLAUDE.md 위험목록에서 그대로 떨어진다:

| 스텁 | 당일에 왜 필요한가 | CLAUDE.md 근거 |
|---|---|---|
| `append-only-원장과-fold` | 재시작 후 점수가 살아있는지 판단 | "로컬 프로세스가 죽어서 점수가 전부 날아가는 게 실제 시나리오다" |
| `websocket과-socket-io-room` | 폰이 안 붙을 때, 빔이 안 뜰 때 | "숙소 와이파이는 복불복" + "이탈/재접속 처리가 실제로 필요하다" |

새 스텁에 `당일`을 붙이려면 **"8시 35분에 이걸 몰라서 못 고치는 상황"을 구체적으로 말할 수
있어야 한다.** 못 하면 `성장`이다. 위험목록에 대응하는 게 없으면 `성장`이다.

### 언제 배우나 — 설문이 창을 연다

학습은 개발 시간을 안 쓴다. Claude가 짓는 동안 다른 터미널에서 병렬로 돈다.
경쟁하는 건 학습과 개발이 아니라 **학습과 설문**이다.

그리고 **설문을 보내면 학습 창이 열린다.** 보내는 순간 남의 답장에 막히고, 그 막힌 시간이
정확히 배우기 좋은 시간이다. 안 보내면 막힐 일이 없으니 학습 창도 안 생긴다.
두 문제가 같은 동작으로 풀린다.

## 무엇이 `decisions/`에 가고 무엇이 코드 주석에 남나

**기준: 답이 파일 2개 이상에 걸치는가.**

- "왜 `phaseEndsAt`이 nullable이냐" → 한 곳. **코드 주석.**
- "왜 Zod냐" → 계약 전체. **ADR.**

그리고 **누가 다르게 했을 법한 것**만 쓴다. "for문을 썼다"는 결정이 아니다.

## 옵시디언 쓰는 법 (처음이면 여기부터)

### 옵시디언이 뭔지부터

**DB도 클라우드도 아니다. 폴더를 읽는 마크다운 뷰어일 뿐이다.**

파일은 이미 여기 다 있다. 옵시디언은 그 위에 링크·검색·그래프를 얹어줄 뿐이고, 내일 지워도
파일은 그대로 남는다. VS Code로도 GitHub로도 읽힌다.

이게 노션과 갈리는 지점이고, 이 프로젝트가 옵시디언을 고른 이유다 — Claude가 파일을 직접
쓸 수 있어서, 문서가 숙제가 아니라 부산물이 된다. 노션이었으면 복붙해야 했다.
**즉 옵시디언은 필수가 아니라 선택적 업그레이드다.** 안 켜도 시스템은 돌아간다.

### 열기 — ★함정 하나 있음★

`Open folder as vault` → **`docs/` 폴더를 고른다.**

**레포 루트(`MT/`)를 고르면 안 된다.** `node_modules/` 안의 파일 수천 개를 인덱싱해서
느려지고 그래프가 쓰레기가 된다. 보관함은 `docs/`다.

처음 열면 "이 보관함을 신뢰하냐" 같은 걸 묻는다. 신뢰하면 된다. 내 파일이다.

### 실제로 쓰는 건 네 개다

| 단축키 | 뭐 하나 |
|---|---|
| `Ctrl+O` | **제일 많이 쓴다.** 노트 이름 몇 글자 치면 바로 이동 |
| `[[링크]]` 클릭 | 따라간다. `Ctrl+클릭`은 옆에 새 창으로 |
| `Ctrl+Shift+F` | 전체 검색 |
| 왼쪽 사이드바 그래프 아이콘 | 노트끼리 어떻게 엮여 있는지 |

나머지 기능은 지금 몰라도 된다. 플러그인도 아직 필요 없다.

### 그래프 뷰가 뭘 보여주나

점이 노트, 선이 `[[링크]]`다. `0002`(Zod 결정)를 클릭하면 거기서 뻗은 개념 3개가 보인다 —
**그 결정을 이해하려면 뭘 알아야 하는지가 그림으로 나온다.**

반대 방향도 된다. 노트 아래 `Backlinks`를 열면 "이 노트를 가리키는 다른 노트"가 뜬다.
`타입-vs-런타임-검증`을 열면 `0002`가 백링크로 잡힌다 — 링크를 한 방향으로만 걸었는데
양방향으로 읽힌다. 이게 옵시디언의 핵심이고, 폴더 구조로는 안 되는 일이다.

### 뭐부터 읽나

1. `concepts/타입-vs-런타임-검증.md` — 유일하게 채워진 노트. 나머지가 어떤 모양이 될지 보여준다
2. `decisions/0003-원칙을-주석이-아니라-타입으로-봉인한다.md` — 이 프로젝트에서 제일 특이한 결정
3. 태그 창에서 `스텁` 클릭 → 끌리는 거 아무거나 → `/learn`으로 질문

## GitHub에 올릴 거면 — 먼저 읽을 것

**퀴즈 문항은 커밋하지 마라.** 설문에서 나올 내용이 "김철수가 1학년 때 ~했다" 같은 실명
개인정보다. 공개 레포에 올리면 동아리원 신상을 인터넷에 뿌리는 것이다.

- 문제 은행은 Postgres에 있고 git에 없다 (CLAUDE.md의 DB 용도 그대로)
- 시드 파일을 만들게 되면 `.gitignore`에 넣는다
- 애매하면 **private 레포**로 시작한다. 코드와 이 문서들은 공개해도 되지만 콘텐츠는 아니다
