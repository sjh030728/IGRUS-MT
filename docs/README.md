# docs/ — 투 트랙 기록

| 폴더 | 뭐가 들어가나 | 누가 쓰나 | 고칠 수 있나 |
|---|---|---|---|
| `decisions/` | **왜 이렇게 짰나.** 이 프로젝트 한정 | 개발 세션이 자동으로 | 예 — 항상 현재 유효한 판단만 담는다. 이력은 git이 가진다 |
| `concepts/` | **여기 쓰인 개발 지식.** 다음 프로젝트에도 씀 | 학습 세션이 질문 받고 | 예 — 현재 최선의 설명 |

섞지 않는다: 결정에 개념 설명이 끼면 길어져 안 읽히고, 개념에 프로젝트 사정이 끼면 다음
프로젝트에서 못 쓴다. `program-ops.md`(진행자용 운영), `survey.md`(설문 문안),
`findings.md`(큐)는 둘 다 아니다.

## 트랙 돌리는 법

**터미널 두 개, 같은 폴더, `claude` 두 번.** 세션이 나뉘면 컨텍스트가 나뉜다.

- **개발 세션:** 평소대로 쓴다. ADR·스텁은 Claude가 시키지 않아도 남긴다 (CLAUDE.md "세션
  기록"). `/logbook`은 수동 강제용.
- **학습 세션:** `/learn <질문>` — **세션 시작에 한 번만.** 이후엔 그냥 물어도 규칙이
  유지된다 (`/clear` 하면 다시 친다). 코드를 절대 수정하지 않고, 답은 `concepts/`에 파일로 남는다.

## 두 큐 — 방향이 반대다

| 큐 | 쌓는 쪽 | 비우는 쪽 | 비우면 |
|---|---|---|---|
| `concepts/` 스텁 | 개발 세션 | 학습 세션 (`/learn`) | `#스텁` 태그를 뗀다 — 노트는 자산으로 남는다 |
| `findings.md` | 학습 세션 | 개발 세션 (CLAUDE.md가 자동 주입) | **항목을 지운다** — 큐지 로그가 아니다 |

**스텁 찾기:** 옵시디언 태그 창의 `스텁`, 또는 `Ctrl+Shift+F` → `tag:#스텁`.
(grep은 코드블록 안 태그까지 세서 틀린다 — 태그 창이 정답이다.)

태그 축은 둘: `#스텁`(본문, 채우면 사라짐) + `영역/…`(frontmatter, 영구 분류).
교집합은 검색이 한다: `tag:#영역/실시간 tag:#스텁`.

규칙 셋 (근거 전문: `decisions/0004`):
- **마감·순서·게이트 없음.** "끌리는 거 아무거나"가 유일한 순서다. 스텁이 밀려도 개발은 안 막힌다
- **문서에 "지금 N개" 금지** — 찾는 법만 적는다. 상한("영역 5개 고정")은 예외
- **`#다시볼것`은 사용자 전용 북마크.** 채워졌는데 아직 소화 못 한 노트에 직접 붙인다.
  Claude는 안 건드린다

## 옵시디언 (선택 — 없어도 시스템은 돈다. 폴더를 읽는 마크다운 뷰어일 뿐이다)

`Open folder as vault` → **`docs/` 폴더를 고른다.** 레포 루트(`MT/`)를 고르면
`node_modules/` 인덱싱으로 느려지고 그래프가 쓰레기가 된다.

쓰는 건 넷: `Ctrl+O`(노트 이동) / `[[링크]]` 클릭 / `Ctrl+Shift+F`(전체 검색) / 태그 창.
백링크 패널을 열면 링크를 한 방향으로만 걸어도 양방향으로 읽힌다.

뭐부터 읽나: 채워진 개념 노트 하나(`path:concepts -tag:#스텁`로 추린다) →
`decisions/` 전부(짧다) → 태그 창에서 끌리는 스텁을 골라 `/learn`.

## GitHub에 올릴 거면

**퀴즈 문항은 커밋하지 마라** — "김철수가 1학년 때 ~했다" 같은 실명 개인정보다.
문제 은행은 Postgres에 있고 git에 없다. 시드 파일을 만들면 `.gitignore`.
애매하면 private 레포. 코드와 이 문서들은 공개해도 되지만 콘텐츠는 아니다.
