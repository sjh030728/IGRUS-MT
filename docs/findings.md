# findings — 개발 큐

**학습 세션이 발견했지만 고칠 수 없는 것들.** 개발 세션이 가져가서 처리하고 **지운다**
(체크만 하지 말 것 — 큐지 로그가 아니다).

**이 파일은 `CLAUDE.md`가 `@docs/findings.md`로 임포트한다** — 매 세션 시작에 내용이
통째로 주입된다(`0014`). 그래서 개발 세션은 이 파일을 "읽으러 갈" 필요가 없고, 여기 뭐가
있으면 이미 보고 있다. **짧게 유지할 이유가 그것이다.**

규칙 전문은 `.claude/commands/learn.md` 규칙 1과 `docs/README.md`("findings가 반대 방향
큐다")에 있다.

---

**비어 있음.** 그게 정상이다 — 쌓이면 학습 세션이 발견한 게 있다는 뜻이고,
비어 있으면 개발 세션이 다 가져갔다는 뜻이다.
