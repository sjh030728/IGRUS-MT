# 0005. 프로그램은 설정이고, 이동은 라운드가 안 도는 동안만이다

- 날짜: 2026-07-16 (단계 2)
- 개념: [[프리플라이트]], [[탈출구]], [[파생-상태와-단일-진실-공급원]]

## 무슨 결정인가

1. **그날의 라인업(program)은 `session.config.json`에 산다.** 세그먼트 = `{id, kind, title, game?}`.
   조 목록과 같은 이유 — 다음 행사는 라인업이 다르고, 7:30에 사람이 고칠 수 있어야 한다.
   config는 게임 목록을 모른다. gameId 실재 검사는 서버 부팅(등록부 대조)이 한다.
2. **부팅 = 프리플라이트 + 첫 세그먼트 진입.** 모든 GAME 세그먼트의 `loadRounds()`를 미리
   돌려보고 실패하면 부팅이 죽는다. 문제은행이 빈 걸 8:03의 SEGMENT_GOTO 거절로 처음
   아는 것과 7:30의 부팅 에러로 아는 것의 차이다.
3. **SEGMENT_GOTO / ROUND_GOTO는 라운드가 안 도는 동안만** (IDLE·REACTION·ABORTED·없음).
   COLLECT 한복판에 이동하면 커밋 안 된 제출이 ★조용히★ 증발한다. 버리는 건 ABORT의
   일이고, ABORT는 시끄럽다 — 빔에 "무효"가 뜬다.
4. **라운드 소진 = 표지.** 마지막 라운드 뒤 ROUND_NEXT는 다음 세그먼트로 안 넘어가고
   점수판 표지로 돌아간다. 세그먼트를 건너는 결정은 항상 사회자의 SEGMENT_GOTO다 (원칙 3).

## 안 고른 것

- **라인업을 코드에** — "재사용 전제"가 거짓말이 된다. 게임을 빼는 게 재빌드가 된다.
- **ROUND_NEXT가 세그먼트를 자동으로 넘어감** — 앱이 진행을 결정하는 세 번째 자동 전이가
  된다. 퀴즈가 끝나면 사회자 멘트 구간이 있어야 하고, 그 길이는 사람이 정한다.
- **진입 시 적재 대신 부팅 시 전부 적재해 캐시** — 재진입(문항 교체 후 다시 seed → goto)이
  안 된다. 프리플라이트는 결과를 버리므로 둘의 장점만 남는다.

## 이 결정이 사는 곳

- `apps/server/session.config.json` — program 섹션 / `core/config.ts` — 스키마
- `core/round.service.ts` — `start()`(프리플라이트) · `gotoSegment/gotoRound`(이동 가드)
- `core/projector.ts` — kind → 빔 모드 파생 (0002의 "모드는 세그먼트에서 파생"의 실물)
- `verify-loop.mjs` §4 — COLLECT 중 이동 거절을 실서버로 확인
