# 0003. 1인 개발로 게임을 7개에서 4개로 줄인다

- 날짜: 2026-07-16 (옛 0004)

## 무슨 결정인가

개발 인원 2 → 1. 게임 7개 개발에서 **4개 개발 / 당일 3개 사용**으로. 슬라이더·두진실은 완전
삭제, 예비는 번역 릴레이 1개. 일정은 주차별 → 단계별(완료 조건 기반)로.

## 왜

인원이 반이면 뭔가는 반드시 떨어지는데, 리허설("게임 1개 추가보다 효과 크다")·리빌 루프(제품
본체)·사운드("필수, 부가기능 아님")·설문(크리티컬 패스)은 못 자른다.
**산술적으로 남는 게 게임뿐이다.**

예비 중 번역 릴레이가 남은 이유: 개발 비용 ~0 (사회자가 입으로 채점하고 ADJUST로 점수),
룰 설명 한 문장, 설문 데이터 불필요. 슬라이더/두진실은 "룰 설명 2분 = 방전 상태에 반응 사망"으로
이미 깎여 있었다. 예비 0개는 안 된다 — 콘솔 필수기능 "예비 게임 즉시 투입"이 죽고,
게임 하나가 5분 만에 끝났을 때 메울 게 없다.

부수 결정:
- **사운드를 단계 1로 끌어올렸다.** 뒤에 있는 건 늦어질 때 잘리는데 문서가 "필수"라고 못
  박았다 — 잘릴 수 없는 것(리빌 루프)과 같은 단계에 묶는다. 마침 사운드는 빔이 phase에서
  파생하는 구조라 서버 코드 0줄이다.
- **주차 → 단계.** 혼자면 어디서 막힐지 예측이 안 된다. 날짜를 박으면 늦어질 때 잘리는 게
  리허설이 된다. 완료 조건은 늦어져도 기준이 안 흔들린다.

## 안 고른 것

- **7개 유지 + 리허설 축소** — 문서가 명시적으로 금지한 방향. 반대로 하지 않는다.
- **계획표는 두고 현실이 자르게 두기** — 자를 시점엔 이미 늦어서, 그때 잘리는 건 마지막에
  있는 것(= 리허설)이다.
