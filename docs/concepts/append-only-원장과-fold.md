# append-only 원장과 fold

> **스텁.** `/learn append-only 원장` 으로 물어보면 채워진다. #스텁

**한 줄:** 값을 덮어쓰는 대신 기록만 계속 쌓고, 필요할 때 처음부터 훑어서(fold) 현재 값을
계산하는 방식. 은행 통장이 이것.

**이 프로젝트에서:**
- `packages/protocol/src/ledger.ts` — `LedgerEntry` 4종(SEED/ROUND/ADJUST/VOID),
  `totals()` 규칙. UPDATE도 DELETE도 없다

**공짜로 따라오는 것 3개:**
1. 프로세스가 죽어도 DB에 기입이 남아 있으니 다시 더하면 복구 끝 (CLAUDE.md 원칙 4)
2. 되감기 = "취소" 기입을 하나 더 추가
3. **연출.** 무효가 기입이면 빔에서 점수가 내려가는 게 보인다

**궁금해할 만한 것:** `fold`가 JS의 `reduce`와 같은 건가? SEED만 `set`이고 나머지가 `add`인
비대칭은 왜? (힌트: 사회자가 7시 58분에 낮 점수를 오타 내면 어떻게 되나)
