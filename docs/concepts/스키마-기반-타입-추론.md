# 스키마 기반 타입 추론 (z.infer)

> **스텁.** `/learn z.infer` 로 물어보면 채워진다. #스텁

**한 줄:** 타입을 손으로 쓰지 않고 스키마에서 자동으로 뽑아내는 것. 정의가 한 벌이라
어긋날 수가 없다.

**이 프로젝트에서:** `packages/protocol/src/` 거의 전 파일. 패턴이 항상 같다:

```ts
export const PlayTap = z.object({ ... });
export type PlayTap = z.infer<typeof PlayTap>;   // ← 이 줄
```

**궁금해할 만한 것:** `const PlayTap`과 `type PlayTap`이 이름이 같은데 왜 충돌 안 하나?
(힌트: TypeScript의 값 네임스페이스와 타입 네임스페이스) `typeof PlayTap`의 `typeof`가
JavaScript의 `typeof`와 같은 건가?
