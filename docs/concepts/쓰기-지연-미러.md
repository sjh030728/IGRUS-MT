# 쓰기 지연 미러 (write-behind mirror)

**한 줄:** 진실은 빠른 곳(메모리)에 두고, 느린 곳(DB)엔 복사본을 ★나중에★ 흘려보내는 것.
쓰는 쪽이 DB의 속도·생사에 볼모로 잡히지 않는다. 반대말은 write-through(둘 다 즉시).

어디서 나왔나: `db.service.ts`의 `mirrorLedger()` + 큐 + `drain()` — 커밋(리빌 직후)이
DB를 기다리지 않고, DB가 죽어도 점수는 계속 돌며, 살아나면 밀린 걸 순서대로 흘린다.
[[큐]]의 생산자/소비자 그림 그대로다.
