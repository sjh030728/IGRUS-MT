# WebSocket과 Socket.IO room

> **스텁.** `/learn websocket` 으로 물어보면 채워진다. #스텁/당일

**한 줄:** 연결을 열어둔 채 서버도 먼저 말을 걸 수 있게 하는 프로토콜. room은 연결들을
그룹으로 묶어 "빔한테만 보내기"를 가능하게 하는 Socket.IO 기능.

**이 프로젝트에서:**
- `packages/protocol/src/events.ts` — `C2S`/`S2C` 이벤트 맵, room 규칙(`display`/`host`/`play`)

**왜 HTTP로는 안 되나:** 사회자가 "공개!"를 누르면 빔과 폰 40대가 동시에 반응해야 한다.
HTTP는 브라우저가 물어봐야 서버가 답하므로 폰이 1초마다 "혹시 공개됐나요?"를 물어야 한다.

**room이 우리한텐 보안 기능인 이유:** 정답은 `host` room에만 간다. CLAUDE.md의
"빔 뷰와 콘솔은 절대 같은 화면에 뜨면 안 된다"가 여기 걸려 있다.

**궁금해할 만한 것:** 이벤트 이름 앞부분(`host:`/`play:`)이 그대로 인증 규칙인 이유는?
왜 "소켓의 room 소속으로 판정하면 안 된다"고 주석에 적혀 있나? (힌트: 클라이언트가 join을
직접 부를 수 있다면?)
